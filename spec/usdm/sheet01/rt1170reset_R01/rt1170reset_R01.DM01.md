# 【rt1170reset_R01.DM01】コマンド受付と応答

## 要求

ホストから受信した 1 行単位の ASCII コマンドを解釈し、成功時と失敗時の応答を一意に返す

### 理由

CLI から安定してリセット操作と状態確認を行えるようにするため

### 説明

コマンド名は大文字小文字を区別せず、前後の空白は無視する。成功応答は CRLF 区切り ASCII で返す

## 仕様

| 種別 | 内容 | 仕様番号 |
| --- | --- | --- |
| 仕様 | `PING` を受信したときは `PONG` を返し、リセット出力は変化させない | 【rt1170reset_R01.DM01-01】 |
| 仕様 | `OFF` を受信したときはリセット出力を Low にして `OK OFF` を返す | 【rt1170reset_R01.DM01-02】 |
| 仕様 | `RESET` または `RESET <ms>` を受信したときはパルス実行後に `OK RESET <実際に適用したms>` を返す | 【rt1170reset_R01.DM01-03】 |
| 仕様 | `BOOTSEL` を受信したときは `OK BOOTSEL` を返した後に RP2040 ROM の USB ブートモードへ遷移する | 【rt1170reset_R01.DM01-04】 |
| 仕様 | 未知のコマンドは `ERR` を返す | 【rt1170reset_R01.DM01-05】 |
| 仕様 | コマンド終端は `CR` または `LF` とし、空行は無視する | 【rt1170reset_R01.DM01-06】 |
