# 【rt1170reset_R06.DM02】検証可能性

## 要求

CLI とファームウェアの主要振る舞いを順を追って確認できるようにする

### 理由

実機接続時に問題が配線、USB、プロトコルのどこにあるかを切り分けるため

### 説明

自動テストが難しいため、実機前提の受け入れ確認手順を重視する

## 仕様

| 種別 | 内容 | 仕様番号 |
| --- | --- | --- |
| 仕様 | `ping` により接続性とプロトコル応答を確認できる | 【rt1170reset_R06.DM02-01】 |
| 仕様 | `off` により出力解除状態を確認できる | 【rt1170reset_R06.DM02-02】 |
| 仕様 | `reset` により RT1170-EVK の再起動を確認できる | 【rt1170reset_R06.DM02-03】 |
| 仕様 | `bootsel` により RP2040 が USB ブートモードへ遷移することを確認できる | 【rt1170reset_R06.DM02-04】 |
